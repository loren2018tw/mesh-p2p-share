## MODIFIED Requirements

### Requirement: 動態連線建議

中控中心 MUST 主動掌握每個下載端的片段擁有狀態，並以片段稀缺性為優先序，自行決定應分配哪個片段給哪個下載端；下載端不得主動請求特定片段。

#### Scenario: 稀缺片段優先配對

- **WHEN** 中控中心執行配對掃描
- **THEN** 中控中心 MUST 先計算每個片段目前被多少端點持有，並從持有者最少的片段開始嘗試配對。

#### Scenario: 找到可配對的來源端與下載端

- **WHEN** 針對目前最稀缺的片段 X，存在持有片段 X 且上傳容量未滿的來源端點 B，以及缺少片段 X 且下載容量未滿的下載端點 C
- **THEN** 中控中心 MUST 指派端點 B 傳送片段 X 給端點 C，不等待端點 C 主動請求。

#### Scenario: 最稀缺片段無可配對則跳至次稀缺

- **WHEN** 針對目前持有者最少的片段，找不到同時符合條件的來源端與下載端
- **THEN** 中控中心 MUST 跳過此片段，改對持有者次少的片段重複嘗試，依此類推。

#### Scenario: 同一片段有多個合格下載端時優先進度落後者

- **WHEN** 多個下載端都缺少同一個稀缺片段，且都符合下載容量限制
- **THEN** 中控中心 MUST 優先將該片段分配給目前已持有片段數較少的下載端。

#### Scenario: 同一輪不重複分配相同片段

- **WHEN** 某片段在本輪配對掃描中已被分配給一個下載端
- **THEN** 中控中心 MUST 不再把同一片段分配給其他下載端，直到下一輪掃描觸發。
